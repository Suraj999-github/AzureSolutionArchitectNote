cloud infrastructure fundamentals

📦 cloud service models (infra focus)

IaaS: virtual machines, storage, networks — you control OS, middleware, apps. Azure manages hardware.

PaaS: managed platform (App Services, SQL DB) — focus on app code, infrastructure abstracted.

SaaS: ready-to-use software (M365).

  • Shared responsibility model: IaaS gives most control and responsibility.
🖥️ virtualization & hypervisors

Physical hardware → hypervisor (Azure uses Hyper-V) → multiple VMs. Enables isolation, flexibility, and efficient resource usage.

  • Azure runs on massive-scale hypervisor clusters.
  • Virtual hardware: vCPU, memory, virtual disks, virtual NICs.
🌍 regions & availability

Region – geographical area with datacenters. Availability Zones – physically separate zones within a region (fault tolerance). Region pairs (e.g., East US & West US) for disaster recovery.

  • Choose region based on latency, compliance, and service availability.
📁 resource hierarchy

Management group → Subscription → Resource group → Resource. Resources are organized in resource groups (logical containers). RBAC and policies apply at each level.

  • ARM (Azure Resource Manager) handles deployment and management.
⚙️ compute infrastructure

Azure VMs: IaaS, choice of OS, sizes, scaling (scale sets). Containers: ACI (simple), AKS (orchestration). Serverless: Azure Functions (event-driven).

  • VM availability sets (update/fault domains) vs Availability Zones.
🌐 networking infrastructure

Virtual Network (VNet): isolated network, subnets, IP addressing. Network Security Groups (NSG): filter traffic. Load balancers: distribute traffic (Azure LB, Application Gateway). VPN Gateway / ExpressRoute: hybrid connectivity.

💾 storage infrastructure

Blob: unstructured data (images, backups). Disk: managed disks for VMs (SSD/HDD). Files: SMB shares. Tiers: hot, cool, archive. Redundancy: LRS, ZRS, GRS.

  • Storage accounts are the base container for Azure Storage services.
🔐 identity & governance

Microsoft Entra ID: identity and access. RBAC: granular permissions. Azure Policy: enforce rules (e.g., only certain VM sizes). Resource locks: prevent accidental deletion.

  • Monitor with Azure Monitor, Advisor, and Service Health.
📘 architect view

· Cloud infrastructure is the foundation: compute, storage, networking delivered as services. Understanding IaaS vs PaaS, virtualization, and resource hierarchy is essential.

📋 key terms

  • IaaS – Infrastructure as a Service
  • Hypervisor – enables VMs
  • VNet – Virtual Network
  • NSG – Network Security Group
  • Availability Zone – isolated datacenter in region
  • ARM – Azure Resource Manager
  • LRS/GRS – storage redundancy

⚙️ infrastructure components

  • Virtual Machines (IaaS)
  • Virtual Networking (VNet, peering)
  • Managed Disks (persistent storage)
  • Load Balancers (traffic distribution)
  • Storage Accounts (blob, files, queues)
  • Azure Resource Groups (logical containers)

🧠 study reminders

  • Shared responsibility – IaaS vs PaaS
  • Region vs Availability Zone vs Region pair
  • Resource groups cannot be nested
  • VMs use virtual disks (page blobs)
  • Azure Policy for compliance, RBAC for access
